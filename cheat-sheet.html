概要

まとめ

  • 【環境構築】
    npm install -g typescript で、インストール。-gオプションをつけることで全てのフォルダでTypeScriptが使用可能になる。
  • 【使い方】
    index.tsのような形でtsファイルを作成後、コンソールで tsc index.ts とすることでコンパイルされ、index.jsファイルが作成される。
  • 【なぜTypeScriptを使うの？】
    1. ドキュメントとしての側面を利用する為
    => 関数を書く際に、引数の型や、戻り値の型を記述する為、ドキュメントとしてわかりやすくなる
    2. Linterとしての側面を利用する為
    => Linterとは、コードが正しいか検証してくれるもの。TypeScriptもコンパイル前にコードチェックを実行してくれる。型の検証もしてくれる点が便利。
    ※ コードエラーの警告が出ていても、jsファイルは作成される
    3. ES5へのコンパイラとしての側面を利用する為
    => ES6に対応していないブラウザもまだ多いが、ES5なら全ても主要なブラウザがサポートしている。
    => ダウングレードしてコンパイルする際は tsc index.ts --target ES2015 のようにしてコンパイルする
    ※ ダウングレードできないものもあるので、 es6 compat table と検索して赤色になっていないか確認が必要
